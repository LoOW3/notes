Son una coleccion de metodos abstractos con propiedades(atributos) CONSTANTES.

Una interfaz solamente puede extender o implementar otras interfaces.

Da a conocer que se debe hacer (metodos) pero sin mostrar su implementacion (solo puede tener metodos abstractos).

Solo puede tener metodos con acceso publico (no pueden ser protected o private).

Solo puede tener "variables" public static final (o sea constantes).

La palabra "abstract" en la definicion de metodos ==no es obligatoria==.

Generalmente las interfaces indican el "PUEDE HACER" de un objeto.

# Ejemplo

Interfaz Figura
```java
package claabstracta;

public interface Figura {

	public double calcularArea(); //declaro el metodo pero no pongo su implementacion
}
```

Interfaz Rotable
```java
package claabstracta;

public interface Rotable {

	public void rotar(); //declaro el metodo pero no pongo su implementacion
}
```

Interfaz Dibujable
```java
package claabstracta;

public interface Dibujable {

	public void dibujar(); //declaro el metodo pero no pongo su implementacion
}
```

Clase cuadrado con implements de Figura, Dibujable
```java
package claabstracta;

public class Cuadrado implements Figura, Dibujable {

	private double lado;
	public Cuadrado() {
	}

	public Cuadrado(double lado) {
		this.lado = lado;
		
	}
	@Override //implementa el mismo metodo pero de forma distinta
	public double calcularArea() {
		double resultado = lado * lado;
		return resultado;
	}
	@Override
	public void dibujar(){
		System.out.printLn("dibujar")
	}
}
```
